# Contributing to Goodnight, From Here

Thank you for considering contributing to this project! This is a deeply personal and artistic work, so we appreciate thoughtful contributions.

## Guidelines

### Code Style
- Follow the existing code structure and patterns
- Use TypeScript for type safety
- Keep components small and focused
- Maintain the premium, minimalist aesthetic

### Design Principles
- Respect the intimate, emotional tone
- Maintain Apple-level attention to detail
- Avoid bright colors or excessive effects
- Ensure animations are smooth and intentional
- Test on mobile devices

### Before Submitting a PR
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages
6. Push to your fork
7. Open a Pull Request

### Pull Request Process
1. Update README.md if needed
2. Describe your changes clearly
3. Link any related issues
4. Ensure no console errors
5. Test on multiple browsers/devices

## Reporting Bugs
Use the bug report template and provide:
- Clear description of the issue
- Steps to reproduce
- Expected behavior
- Browser/device information

## Suggesting Features
Use the feature request template and explain:
- Why the feature matters
- How it enhances the experience
- How it fits the overall vision

## Questions?
Open a discussion or issue to talk about ideas first.

---

**Remember**: This is an art project first. Every change should serve the emotional narrative.
